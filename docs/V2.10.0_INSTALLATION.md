# NCR Suite V2.10.0 — Installation

Cette version s’installe sur la **V2.9.2**.

## Ordre obligatoire

1. Supabase SQL
2. GitHub
3. Cloudflare Pages
4. Fermeture et réouverture de la PWA

## 1. Supabase

Dans le projet Supabase de NCR Suite, ouvre **SQL Editor**, crée une nouvelle requête, copie tout le contenu de :

```text
supabase/migrations/052_restaurant_recipes_stock_profitability.sql
```

Clique sur **Run**. Le résultat attendu est `Success`.

La migration :

- crée les fiches recettes et leurs ingrédients avec un enregistrement transactionnel ;
- crée l’historique des mouvements de stock ;
- ajoute les allergènes aux références de stock ;
- installe le calcul du coût matière ;
- installe le déstockage des articles servis ;
- installe la déduction des pertes ;
- répare les droits des offres Restauration.

Elle ne supprime aucune commande, réservation, table, plat ou stock existant.

## 2. GitHub

Charge tous les autres fichiers du patch dans le dépôt NCR Suite en conservant exactement l’arborescence.

Ne crée pas de dossier supplémentaire autour de `src`, `public` ou `supabase`.

## 3. Cloudflare

Vérifie qu’un nouveau déploiement est créé pour le commit V2.10.0 et qu’il termine avec le statut **Success**.

## 4. PWA

Ferme complètement NCR Suite puis rouvre-la. Le nouveau cache est :

```text
ncr-suite-shell-v2.10.0-restaurant-recipes
```

## Tests essentiels

### Fiche recette

1. Ajoute au moins deux produits dans **Stocks & fournisseurs**.
2. Ouvre **Fiches recettes**.
3. Sélectionne un plat.
4. Ajoute les ingrédients, les quantités et la méthode.
5. Enregistre la fiche.
6. Vérifie le coût et la marge avec un compte propriétaire ou administrateur en offre Professionnelle.

### Allergènes

1. Déclare un allergène sur un ingrédient du stock.
2. Enregistre ou recharge la fiche recette.
3. Clique sur **Appliquer à la carte**.
4. Vérifie la présence de l’allergène sur le plat et le menu public.

### Déstockage automatique

1. Utilise une entreprise en offre Professionnelle.
2. Ouvre une commande et ajoute un plat possédant une fiche recette.
3. Envoie l’article en Cuisine.
4. Passe-le à **En préparation**, **Prêt**, puis **Servi**.
5. Vérifie que les quantités d’ingrédients ont été déduites une seule fois.
6. Recharge la page : aucune deuxième déduction ne doit se produire.
7. Corrige la quantité d’un article déjà servi : l’ancienne consommation doit être rétablie puis la nouvelle appliquée une seule fois.

### Pertes

1. Ouvre **Stocks & fournisseurs**.
2. Déclare une perte en choisissant un produit du stock.
3. Vérifie que sa quantité est déduite et qu’un mouvement apparaît dans la traçabilité.

## Répartition des offres

- **Découverte** : création et gestion des fiches recettes.
- **Essentielle** : consultation des fiches par l’équipe Cuisine.
- **Professionnelle** : coûts matière, marges, déstockage automatique et traçabilité.
- **Métier** : mêmes fonctions avec capacités et intégrations personnalisées.
