# NCR Suite V2.20.0

## Contenu

La V2.20.0 finalise la stabilisation multi-métiers et ajoute les modules Formation à la carte.

L’utilisateur Formation peut consulter les modules compatibles avec sa formule, connaître leur prix et demander leur activation ou leur retrait. Le total mensuel projeté est comparé à l’offre supérieure. Lorsque cette offre devient moins chère ou équivalente, NCR Suite le signale avant l’activation.

Le super administrateur traite les demandes depuis l’onglet `Abonnements` et peut associer un lien Qonto distinct à chaque module. Sans lien actif, la demande suit le circuit de validation manuelle.

## Déploiement

1. Exécuter `supabase/migrations/080_final_stabilization_training_modules.sql`.
2. Vérifier `platform_release_state`.
3. Envoyer les fichiers du patch sur GitHub.
4. Attendre le build Cloudflare.
5. Actualiser le cache PWA puis exécuter les tests fonctionnels.

Ne pas rejouer les migrations précédentes. Aucune Edge Function n’est modifiée.

## Règles tarifaires

- Le prix de base de la formule n’est pas modifié par une demande.
- Seuls les modules actifs s’ajoutent au total mensuel.
- Les demandes en attente apparaissent dans le total projeté.
- CRM est requis avant Facturation.
- Facturation est requise avant BPF.
- Une montée de formule désactive les suppléments désormais inclus.
- Les opérations sont historisées dans les journaux d’audit.

## Vérifications

- Le build doit annoncer `ncr-suite@2.20.0`.
- Le cache attendu est `ncr-suite-shell-v2.20.0-final-stabilization`.
- `npm run audit:phase1`, `npm run test:critical` et `npm run test:release` doivent réussir.
- La supervision NCR doit afficher le bloc `Préparation V2.20`.
