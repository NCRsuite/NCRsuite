# NCR Suite V2.18.0 — Formation · Facturation et encaissements

## Objet du patch

La V2.18.0 transforme les dossiers commerciaux Formation déjà validés en factures opérationnelles. Elle conserve les clients, financeurs, sessions, automatisations et calculs BPF existants.

Fonctions ajoutées :

- facture brouillon depuis un devis, une convention ou un contrat accepté ;
- facturation partielle, client ou financeur, sans dépasser le solde du dossier ;
- numéro définitif attribué uniquement à l’émission ;
- PDF facture ou avoir avec identité du vendeur et du payeur figée ;
- envoi Brevo avec PDF joint ;
- encaissements partiels ou complets et historique des règlements ;
- échéances, retards et relances automatiques ;
- réglages de paiement, banque, TVA et relances ;
- BPF alimenté par les factures et avoirs émis lorsqu’ils existent.

## Ordre de déploiement

1. Dans le SQL Editor Supabase, exécuter uniquement `supabase/migrations/078_training_billing_collections.sql`.
2. Vérifier que la transaction se termine sans erreur et que `platform_release_state.database_version` vaut `2.18.0`.
3. Appliquer tous les fichiers du patch V2.18.0 sur GitHub en conservant leur emplacement.
4. Redéployer la fonction Edge `process-email-queue`.
5. Lancer le build Cloudflare. La ligne attendue est `ncr-suite@2.18.0 build`.
6. Ouvrir l’application et laisser le service worker activer `ncr-suite-shell-v2.18.0-training-billing-collections`.
7. Fermer puis rouvrir la PWA, ou vider une fois les données du site si l’ancien cache reste affiché.

Ne pas rejouer les migrations `001` à `077` : elles constituent la base déjà déployée.

## Première configuration

1. Ouvrir `Facturation`, puis l’onglet `Réglages`.
2. Vérifier le préfixe, le délai de paiement et les pénalités.
3. Renseigner la mention d’exonération si des factures sont émises avec une TVA à 0 %.
4. Ajouter l’IBAN et le BIC à afficher sur les PDF.
5. Régler le délai et le nombre maximal de relances.
6. Vérifier dans `CRM & commercial` l’adresse et l’e-mail des clients et financeurs.

## Test fonctionnel

1. Choisir un dossier commercial accepté ou signé.
2. Créer un brouillon avec un montant inférieur ou égal au reste facturable.
3. Télécharger le PDF : le filigrane `BROUILLON` doit être présent.
4. Émettre la facture : un numéro `FAC-AAAA-000001` doit être attribué.
5. Envoyer la facture : l’e-mail doit entrer dans `email_outbox` et contenir le PDF.
6. Enregistrer un règlement partiel puis le solde.
7. Créer un avoir depuis une facture émise et vérifier son numéro `AV-AAAA-000001`.
8. Actualiser le BPF de l’exercice : les factures et avoirs doivent remplacer les montants commerciaux.
9. Tester l’affichage ordinateur et mobile après le build Cloudflare.

## Relances automatiques

La fonction Edge appelle `queue_due_training_invoice_reminders` à chaque passage. Une relance nécessite :

- une facture envoyée avec un PDF stocké ;
- un solde positif et une échéance dépassée ;
- les relances activées pour l’organisme ;
- une adresse e-mail valide dans l’identité figée du payeur.

Chaque relance possède une clé de déduplication et reste visible dans la file d’e-mails.

## Portée réglementaire

Le patch produit des factures PDF et organise leur suivi. Il ne transmet pas encore de facture électronique structurée via une plateforme agréée et ne revendique pas la compatibilité Factur-X. Cette intégration fera l’objet d’une version dédiée.
