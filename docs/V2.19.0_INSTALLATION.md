# NCR Suite V2.19.0 — Formation · Qualiopi, conformité et preuves

## Objet du patch

La V2.19.0 ajoute un dossier qualité opérationnel au parcours Formation. Elle conserve les automatisations, le CRM, le BPF, la facturation et tous les documents existants.

Le module fournit :

- le suivi des 7 critères et 32 indicateurs du référentiel qualité ;
- l’applicabilité, le statut, le responsable, l’échéance et les actions par indicateur ;
- la réutilisation automatique des preuves déjà créées dans NCR Suite ;
- le dépôt de preuves complémentaires avec date de validité ;
- le calendrier et le résultat des audits ;
- une photographie du dossier lors de la clôture d’un audit ;
- les exports de préparation PDF et CSV.

Le module est un outil de préparation et de traçabilité. La décision de conformité appartient à l’organisme certificateur.

## Ordre de déploiement

1. Dans le SQL Editor Supabase, exécuter uniquement `supabase/migrations/079_training_quality_compliance.sql`.
2. Vérifier que la transaction se termine sans erreur.
3. Contrôler dans `platform_release_state` :
   - `database_version = 2.19.0`
   - `expected_frontend_version = 2.19.0`
   - `expected_pwa_cache = ncr-suite-shell-v2.19.0-training-quality-compliance`
4. Appliquer tous les fichiers du patch V2.19.0 sur GitHub en conservant leur emplacement.
5. Lancer le build Cloudflare. La ligne attendue est `ncr-suite@2.19.0 build`.
6. Ouvrir l’application et laisser le service worker activer le nouveau cache.
7. Fermer puis rouvrir la PWA. Si nécessaire, vider une fois les données du site.

Ne pas rejouer les migrations `001` à `078`. Aucune fonction Edge n’est modifiée par cette version.

## Première ouverture

À l’ouverture de `Qualiopi & conformité`, NCR Suite crée les 32 indicateurs manquants puis synchronise :

- les programmes avec l’indicateur 1 ;
- les convocations avec l’indicateur 9 ;
- les attestations avec l’indicateur 11 ;
- les documents administratifs avec l’indicateur 18 ;
- les supports pédagogiques avec l’indicateur 19 ;
- les évaluations complétées avec l’indicateur 30.

La synchronisation est idempotente : relancer `Actualiser` ne crée pas de doublons.

## Droits

- Les propriétaires, administrateurs et responsables peuvent consulter et mettre à jour les indicateurs et les preuves.
- La planification et la clôture d’un audit sont réservées aux propriétaires et administrateurs.
- Les tables sont cloisonnées par entreprise avec RLS.
- Les écritures passent par des fonctions Supabase contrôlées.
- Les offres Formation Professionnelle reçoivent automatiquement le module.
- Pour une offre Métier configurée à la carte, activer `training_quality` dans les modules de l’entreprise.

## Tests après déploiement

1. Ouvrir un espace Formation Professionnelle puis vérifier la rubrique `Qualiopi & conformité`.
2. Vérifier la présence de 32 indicateurs répartis dans 7 critères.
3. Cliquer sur `Actualiser` et contrôler l’apparition des preuves issues des documents existants.
4. Ouvrir un indicateur sans preuve, ajouter un PDF puis enregistrer le statut `Maîtrisé`.
5. Vérifier qu’un indicateur ne peut pas être déclaré `Maîtrisé` sans preuve active.
6. Définir une date d’expiration sur une preuve puis contrôler son affichage.
7. Planifier un audit interne, passer son statut à `En préparation`, choisir un résultat puis le terminer.
8. Exporter le dossier PDF et le CSV.
9. Tester l’écran sur ordinateur puis mobile : aucun texte ou bouton ne doit se chevaucher.
10. Vérifier qu’un utilisateur d’une autre entreprise ne lit aucune donnée du dossier.

## Référence réglementaire

Le module reprend la structure des 7 critères et 32 indicateurs du référentiel national de certification qualité. Les règles applicables, les preuves attendues et les adaptations dépendent de l’activité de l’organisme et de la version du guide de lecture utilisée lors de l’audit.
