# NCR Suite V2.12.2 — Portail client Nettoyage

## Prérequis

La V2.12.1 doit être installée et validée. NCR Academy reste un projet séparé et ne doit pas être modifié.

## Installation

1. Exécuter `supabase/migrations/063_cleaning_client_portal.sql` après la migration 062.
2. Redéployer l’Edge Function `process-email-queue` afin d’activer le modèle `cleaning_client_portal_invitation`.
3. Redéployer l’Edge Function `admin-delete-organization` afin d’inclure le bucket `cleaning-client-documents` dans la suppression sécurisée.
4. Ajouter ou remplacer les autres fichiers dans GitHub en conservant leur arborescence.
5. Vérifier que le déploiement Cloudflare passe les audits, la compilation TypeScript et le build Vite.
6. Fermer complètement la PWA puis la rouvrir pour charger `ncr-suite-shell-v2.12.2-cleaning-client-portal`.

## Validation fonctionnelle

1. Depuis une entreprise Nettoyage disposant de l’offre Métier, ouvrir **Portail clients**.
2. Sélectionner un client actif et envoyer une invitation.
3. Accepter l’invitation avec l’adresse e-mail ciblée.
4. Vérifier les permissions séparément : interventions, rapports, anomalies, qualité, documents et messages.
5. Publier puis ouvrir un document.
6. Envoyer un message dans les deux sens et vérifier la disparition des compteurs non lus.
7. Suspendre puis réactiver l’accès client.

La migration est incrémentale et ne supprime aucune donnée métier existante.
