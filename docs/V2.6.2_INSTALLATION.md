# Installation NCR Suite V2.6.2

Base requise : **V2.6.1**.

## 1. Supabase

Exécuter entièrement `supabase/migrations/039_security_operations_finalization.sql` dans le SQL Editor et attendre le message **Success**.

Cette migration ajoute :

- la prise et la fin de poste rattachées à la vacation ;
- la clôture de la main courante ;
- la régularisation des opérations par le QG ;
- la correction motivée des minutes facturées ;
- la facturation définitive basée sur les heures planifiées par défaut ;
- les contrôles et journaux d’audit associés.

Ne relancer aucune migration antérieure.

## 2. Edge Functions

Aucun redéploiement n’est obligatoire. Le correctif d’envoi des devis se trouve dans l’application web et utilise la fonction `send-security-document` déjà déployée.

Ne pas modifier `process-email-queue`, `send-security-document`, `setup-push-notifications` ou `process-push-notifications`.

## 3. GitHub et Cloudflare

Déposer tout le contenu du patch à la racine de la V2.6.1, conserver l’arborescence, accepter les remplacements puis attendre **Success** dans Cloudflare.

## 4. Cache PWA

Fermer complètement NCR Suite puis la rouvrir. Le cache doit passer en V2.6.2.

## 5. Tests ciblés

1. Envoyer un devis par e-mail et vérifier sa réception.
2. Ouvrir une fiche agent depuis la page Agents et contrôler ses totaux et ses missions.
3. Terminer une vacation puis vérifier que la facture reprend les heures planifiées.
4. Appliquer une correction de durée avec un motif et actualiser le calcul.
5. Tester les boutons Prendre mon poste et Terminer mon poste avec un compte agent.
6. Régulariser une vacation depuis Dossiers vacations avec un compte QG.
7. Depuis un compte Agent Professionnel, prendre le poste et vérifier la première position GPS, puis ouvrir GPS / PTI, utiliser Tester ma position et démarrer le mode vacation.
