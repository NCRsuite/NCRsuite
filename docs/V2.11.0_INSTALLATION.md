# NCR Suite V2.11.0 — Administration NCR & Exploitation SaaS

Cette version s’installe sur la base stable V2.10.1.

## Ordre obligatoire

1. Exécuter `supabase/migrations/054_saas_admin_onboarding.sql` dans le SQL Editor Supabase.
2. Vérifier le résultat `Success`.
3. Ajouter ou remplacer les autres fichiers du patch dans GitHub en conservant leur arborescence.
4. Vérifier le déploiement Cloudflare.
5. Fermer complètement la PWA puis la rouvrir.

## Cache PWA

`ncr-suite-shell-v2.11.0-saas-admin`

## Vérifications rapides

### Administration NCR

- Ouvrir `/administration-ncr` avec le compte super-administrateur.
- Vérifier le cockpit, les indicateurs et le portefeuille par métier.
- Ouvrir la rubrique Entreprises et sélectionner une entreprise.
- Vérifier l’indicateur de santé, l’onboarding et les tickets ouverts.
- Ouvrir Support puis Journal.

### Onboarding

- Créer un nouveau compte sans entreprise.
- Suivre les quatre étapes de configuration.
- Vérifier que l’espace créé utilise le métier choisi.
- Vérifier dans l’administration NCR que l’onboarding apparaît comme terminé.

### Assistance client

- Depuis une entreprise, ouvrir `Assistance NCR`.
- Créer une demande.
- Vérifier qu’elle apparaît dans l’administration NCR > Support.
- Modifier son statut et ajouter une note interne.
- Vérifier que la note apparaît dans l’espace de l’entreprise.

## Remarques

- Les entreprises existantes sont automatiquement considérées comme déjà configurées afin de ne pas fausser les indicateurs.
- Les nouvelles entreprises démarrent avec un onboarding à compléter.
- Les tickets sont accessibles uniquement via les fonctions sécurisées prévues par la migration.
