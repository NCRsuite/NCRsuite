# NCR Suite V2.20.1

## Ce qui change

Les responsables d’un organisme de formation voient maintenant les modules disponibles même lorsqu’ils ne sont pas inclus dans leur formule. Les rubriques restent présentes dans les menus ordinateur et mobile, accompagnées d’un cadenas et du nom de l’offre nécessaire.

Le clic n’ouvre jamais les données du module verrouillé. Il affiche une fiche de présentation avec un bouton vers `Mon abonnement`. La page d’abonnement se positionne automatiquement sur le module concerné et met sa carte en évidence.

Les modules déjà inclus ou activés à la carte continuent de fonctionner sans étape supplémentaire. Pour une offre Métier, un module désactivé renvoie vers la configuration contractuelle.

## Déploiement

1. Exécuter uniquement `supabase/migrations/081_training_locked_module_navigation.sql`.
2. Vérifier la ligne `platform_release_state`.
3. Envoyer tous les fichiers du patch sur GitHub en conservant leurs chemins.
4. Attendre la réussite du build Cloudflare.
5. Renouveler le cache PWA et réaliser les tests fonctionnels.

La migration 080 étant déjà déployée avec la V2.20.0, elle ne doit pas être rejouée. Aucune Edge Function n’est modifiée.

## Vérifications fonctionnelles

- Offre Découverte : `Personnalisation` est visible sous cadenas et pointe vers son module Essentielle.
- Offre Découverte ou Essentielle : CRM, Facturation, BPF, Qualiopi, Dossiers, Évaluations, Établissements et Accès équipe restent visibles sous cadenas lorsqu’ils ne sont pas activés.
- Le menu mobile présente les mêmes cadenas que le menu ordinateur.
- La fiche verrouillée ne charge pas la page métier.
- Le bouton de montée en gamme ouvre et met en évidence la bonne carte.
- Un module activé à la carte ouvre directement sa page.
- Une offre Professionnelle conserve ses accès inclus.
- Une offre Métier renvoie vers sa configuration lorsqu’un module n’est pas inclus.

## Contrôles techniques

- Le build doit annoncer `ncr-suite@2.20.1`.
- Le cache attendu est `ncr-suite-shell-v2.20.1-training-locked-navigation`.
- `npm run audit:phase1`, `npm run test:critical`, `npm run test:release` et `npm run build` doivent réussir.
