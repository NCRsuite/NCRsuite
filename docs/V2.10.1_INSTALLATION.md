# NCR Suite V2.10.1 — Correctif déstockage automatique

## Ordre d'installation

1. Exécuter `supabase/migrations/053_restaurant_stock_consumption_repair.sql` dans Supabase.
2. Envoyer les autres fichiers du patch sur GitHub en conservant leur arborescence.
3. Vérifier le déploiement Cloudflare.
4. Fermer complètement la PWA puis la rouvrir.

## Rattraper le test déjà effectué

Après le déploiement, ouvrir **Restauration → Stocks & fournisseurs** puis cliquer sur **Synchroniser les plats servis**. La fonction analyse les articles servis des 7 derniers jours et crée uniquement les mouvements manquants.

## Contrôle

Lorsqu’un article est marqué **Servi** dans l’écran Cuisine, NCR Suite affiche désormais :

- le nombre d’ingrédients déduits ;
- ou la raison précise de l’absence de déstockage : fiche recette absente, aucun ingrédient déductible, plat dissocié de la carte ou fonction non incluse.

Le mécanisme reste idempotent : relancer la synchronisation ou recliquer sur un statut déjà servi ne déduit jamais deux fois le même ingrédient.
