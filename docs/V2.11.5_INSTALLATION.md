# NCR Suite V2.11.5 — Matrice d’accès, RLS et pages publiques

## Ordre d’installation

1. Exécuter `supabase/migrations/058_access_matrix_rls_public_hardening.sql` dans Supabase.
2. Ajouter les autres fichiers du patch sur GitHub en conservant leur arborescence.
3. Vérifier le déploiement Cloudflare.
4. Fermer puis rouvrir complètement la PWA.

## Contrôles principaux

- Une entreprise ne doit jamais ouvrir une page appartenant à un autre métier en saisissant son URL.
- Un employé ne doit pas ouvrir Abonnement, Accès équipe, Personnalisation ou Configuration Métier.
- Un manager ne doit pas modifier l’abonnement ni les accès de l’entreprise.
- Une offre Sécurité Découverte sans module Agent doit afficher le verrouillage de l’espace terrain.
- Administration NCR > Diagnostic doit afficher le contrôle « Sécurité des accès ».
- Les pages publiques Coiffure, Restauration et Formation doivent continuer à fonctionner pour les entreprises actives et autorisées.
