# NCR Suite V2.12.0 — Installation

Cette version ouvre la phase 3 avec le **Portail client Sécurité**. Elle s’installe sur la base stable V2.11.6.

## Fonctions livrées

- rubrique **Portail clients** dans le métier Sécurité ;
- invitations sécurisées des donneurs d’ordre ;
- accès Responsable client ou Consultation ;
- permissions séparées pour le planning, la main courante, les rondes, les documents et la messagerie ;
- portail externe à l’adresse `/espace-client-securite` ;
- suivi des sites, missions, événements et rondes autorisés ;
- documents privés jusqu’à 15 Mo ;
- messagerie client/prestataire ;
- suspension et réactivation des accès ;
- journalisation des actions sensibles et limitation anti-abus.

Le portail est inclus dans les offres **Sécurité Professionnelle** et **Métier**.

## Ordre d’installation

### 1. Supabase — migration SQL

Exécuter uniquement :

```text
supabase/migrations/060_security_client_portal.sql
```

Les migrations 054 à 059 ne doivent pas être rejouées.

Le SQL crée les tables du portail, les fonctions sécurisées, le compartiment privé `security-client-documents`, les politiques RLS et le nouveau modèle de file e-mail.

### 2. Supabase — Edge Function e-mail

La fonction suivante a été modifiée :

```text
supabase/functions/process-email-queue/index.ts
```

Elle doit être redéployée pour que les invitations au portail utilisent le nouveau modèle d’e-mail.

Depuis le tableau de bord Supabase :

1. ouvrir **Edge Functions → process-email-queue** ;
2. remplacer intégralement son code par le fichier du patch ;
3. déployer la fonction ;
4. conserver **Verify JWT désactivé**, comme sur la version actuellement utilisée.

La même opération peut être réalisée avec Supabase CLI :

```bash
supabase functions deploy process-email-queue --no-verify-jwt
```

Les secrets Brevo existants restent inchangés. L’adresse d’expédition configurée peut rester `contact@ncr-suite.fr`.

### 3. GitHub

Déposer les autres fichiers du patch dans GitHub en conservant exactement leur arborescence.

### 4. Cloudflare

Vérifier que le build exécute puis valide :

```text
Audit NCR Suite : contrôles statiques validés.
Parcours critiques NCR Suite : validation statique réussie.
```

### 5. PWA

Fermer complètement NCR Suite puis la rouvrir.

Nouveau cache :

```text
ncr-suite-shell-v2.12.0-security-client-portal
```

## Configuration Auth recommandée

Dans **Supabase → Authentication → URL Configuration**, vérifier que le domaine de production NCR Suite est autorisé dans les URL de redirection. Le lien de confirmation d’un nouveau compte doit pouvoir revenir sur :

```text
/client-securite/invitation/*
```

## Tests conseillés

1. Ouvrir une entreprise Sécurité en offre Professionnelle ou Métier.
2. Aller dans **Portail clients**.
3. Sélectionner un client actif puis inviter son adresse e-mail.
4. Vérifier la réception de l’e-mail Brevo.
5. Ouvrir le lien, créer le compte client et accepter l’invitation.
6. Vérifier que le client ne voit que les sites et données de sa propre fiche.
7. Désactiver une permission puis vérifier que la rubrique correspondante disparaît.
8. Publier un PDF et vérifier son ouverture depuis le portail.
9. Envoyer un message dans les deux sens.
10. Suspendre l’accès puis vérifier que le portail n’est plus accessible.
11. Tester une entreprise Sécurité Découverte ou Essentielle : la page doit proposer l’offre Professionnelle sans ouvrir le portail.

## Important

- Les documents sont stockés dans un compartiment Supabase privé.
- Les liens de téléchargement sont temporaires.
- Les comptes clients ne deviennent jamais membres internes de l’entreprise de sécurité.
- Aucune donnée de NCR Academy n’est modifiée.
