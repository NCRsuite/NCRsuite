# NCR Suite V2.7.1 — Installation

Cette version complète le métier Nettoyage avec les protocoles par site, la planification récurrente, les checklists terrain et la rentabilité par chantier.

## 1. Supabase

Exécuter entièrement la migration suivante dans l’éditeur SQL Supabase :

```text
supabase/migrations/044_cleaning_protocols_profitability.sql
```

La migration doit être exécutée après `043_cleaning_pack_core.sql`.

Elle :

- enrichit les droits des offres Nettoyage ;
- crée les protocoles et leurs tâches ;
- crée les programmations récurrentes ;
- ajoute les coûts horaires confidentiels ;
- ajoute les coûts de consommables aux interventions ;
- sécurise les accès avec RLS ;
- installe les fonctions de génération, de checklist et de calcul de marge.

## 2. GitHub

Copier les fichiers du patch à la racine du dépôt en conservant leur arborescence exacte. Les fichiers existants doivent être remplacés et les nouveaux fichiers ajoutés.

## 3. Cloudflare

Après le commit sur la branche de production :

1. vérifier qu’un nouveau déploiement Cloudflare apparaît ;
2. vérifier que le build se termine avec succès ;
3. ne pas utiliser un ancien déploiement mis en cache.

## 4. Cache PWA

Fermer complètement NCR Suite puis la rouvrir. Le nouveau cache est :

```text
ncr-suite-shell-v2.7.1-cleaning-protocols
```

## 5. Tests recommandés

### Offre Découverte

- créer un protocole pour un site ;
- ajouter plusieurs tâches ;
- créer une récurrence ;
- vérifier la génération des huit prochaines semaines ;
- relancer la génération et vérifier l’absence de doublons ;
- vérifier que la rubrique Rentabilité est verrouillée.

### Offre Essentielle

- planifier une intervention avec un protocole ;
- démarrer l’intervention depuis l’espace agent ;
- cocher les tâches ;
- vérifier le blocage du départ si une tâche obligatoire reste incomplète ;
- vérifier la demande de photo après lorsque le protocole l’impose ;
- télécharger le rapport PDF et contrôler la checklist.

### Offre Professionnelle

- renseigner le coût horaire d’un agent ;
- renseigner les consommables d’une intervention ;
- vérifier le chiffre, les coûts, la marge et le taux de marge par site ;
- vérifier qu’un compte Chef d’équipe ne peut pas accéder à la rentabilité.

## 6. Retour arrière

Ne pas supprimer la migration après son exécution. En cas de problème d’interface, restaurer uniquement les fichiers GitHub de la V2.7.0 et conserver les nouvelles tables Supabase ; elles n’affectent pas les autres métiers.
