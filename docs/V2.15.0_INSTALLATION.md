# Installation — NCR Suite V2.15.0

## Prérequis

- Base applicative : **V2.14.1 validée**.
- Correctif PDF commercial Formation déjà appliqué.
- Edge Function `process-email-queue` déjà déployée et planifiée.

## 1. Migration Supabase

Exécuter intégralement :

```text
supabase/migrations/070_training_unified_workflow.sql
```

Cette migration ajoute le profil unique de l’organisme, complète les fiches formations, relie les propositions commerciales, crée la conversion proposition → session et sécurise la validation avec mise en file des convocations.

## 2. Patch GitHub

Copier tous les fichiers du patch en conservant leur chemin. Aucun fichier de la base V2.14.1 ne doit être supprimé.

## 3. Déploiement Cloudflare

Le build attendu exécute :

```text
node scripts/phase1-static-audit.mjs
node scripts/phase1-critical-flows.mjs
tsc -b
vite build
```

Le déploiement doit terminer en vert.

## 4. Cache PWA

Fermer complètement NCR Suite, puis la rouvrir. Le cache attendu est :

```text
ncr-suite-shell-v2.15.0-training-workflow
```

## 5. Parcours de validation

1. Ouvrir **Profil organisme** et compléter les informations essentielles.
2. Créer une **Formation complète**.
3. Associer au moins un formateur actif.
4. Depuis la formation, cliquer sur **Créer une proposition**.
5. Créer le devis, la convention ou le contrat.
6. Passer le document à « Envoyé », puis importer le retour signé.
7. Cliquer sur **Créer la session**.
8. Sélectionner le formateur, les dates, le lieu et les stagiaires validés.
9. Dans **Parcours Formation**, valider la session.
10. Vérifier que les convocations sont générées et envoyées par le processeur existant.

## Périmètre de la V2.15.0

Cette version installe le socle du parcours unifié. La refonte graphique commune des PDF et les envois Brevo des documents commerciaux seront ajoutés en V2.15.1. Les évaluations automatisées, relances, attestations et la clôture complète seront ajoutées en V2.15.2.
