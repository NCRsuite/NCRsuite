# NCR Suite V2.15.3 — Formation · Intégrité des automatisations

Base requise : **NCR Suite V2.15.2 validée**.

## Objectif

Cette version ne refait pas les automatisations déjà validées en V2.15.2. Elle rend le dépôt autoportant pour les documents automatiques, empêche la planification d’une session Formation sans validation officielle, et synchronise la version frontend/cache.

## Ordre d’installation

1. Exécuter `supabase/migrations/074_training_automation_integrity.sql` dans Supabase.
2. Appliquer le patch GitHub.
3. Redéployer `process-email-queue` si l’Edge Function n’est pas automatiquement publiée.
4. Lancer le build Cloudflare.
5. Vider/actualiser le cache PWA après activation du service worker V2.15.3.
6. Contrôler `npm run audit:phase1`, `npm run test:critical`, puis `npm run build`.

## Points couverts

- Table `training_document_jobs` et fonctions service-role associées.
- Colonnes documentaires `generated_automatically`, `automation_key`, `generated_at`, `emailed_at`.
- Garde SQL `guard_training_session_validation`.
- Rapport `training_automation_integrity_report`.
- Écran Sessions raccordé au workflow officiel `validate_training_session_workflow`.
- Cache `ncr-suite-shell-v2.15.3-training-automation-integrity`.

NCR Academy n’est pas concerné par ce patch.
