# NCR Suite V2.8.3 — Anti-surréservation des tables

## Installation

1. Exécuter `supabase/migrations/048_restaurant_reservation_conflict_guard.sql` dans le SQL Editor Supabase.
2. Ajouter les autres fichiers du patch dans GitHub en conservant leur arborescence.
3. Vérifier le déploiement Cloudflare.
4. Fermer puis rouvrir complètement la PWA.

Cache PWA : `ncr-suite-shell-v2.8.3-reservation-lock`.

## Comportement

- Une table trop petite, indisponible ou déjà occupée sur un créneau n'est plus proposée dans le formulaire interne.
- Dès qu’un plan contient des tables, une réservation interne sans table est refusée afin d’éviter toute surréservation.
- La durée choisie est prise en compte : deux réservations successives restent possibles si leurs horaires ne se chevauchent pas.
- Supabase refuse toute double réservation, y compris lors de deux créations simultanées.
- Une réservation en ligne attribue automatiquement la plus petite table adaptée et bloque le créneau tant qu'elle reste en attente, confirmée ou installée.
- Une réservation annulée, terminée ou marquée absente libère immédiatement le créneau.
- Si aucune table n'a encore été configurée, la réservation publique reste disponible avec attribution manuelle ultérieure.

## Tests

1. Créer une réservation Table 1 à 19 h pour 2 heures.
2. Créer une seconde réservation à 20 h : Table 1 ne doit plus être proposée.
3. Créer une réservation à 21 h : Table 1 doit redevenir disponible.
4. Annuler la première réservation : Table 1 doit être de nouveau proposée à 20 h.
5. Tester le lien public sur un créneau complet : le bouton d'envoi doit être bloqué.
