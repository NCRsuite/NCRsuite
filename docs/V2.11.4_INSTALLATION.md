# NCR Suite V2.11.4 — Phase 1 · Robustesse globale et performance

Cette version constitue le premier lot de la phase 1 « Audit et fiabilité globale ».
Elle ne modifie ni les données Supabase, ni les offres, ni les droits métier.

## Installation

1. Aucun SQL Supabase à exécuter.
2. Remplacer ou ajouter les fichiers du patch dans GitHub en conservant leur arborescence.
3. Vérifier le déploiement Cloudflare.
4. Fermer complètement la PWA puis la rouvrir.

Cache PWA : `ncr-suite-shell-v2.11.4-global-reliability`.

## Contrôles recommandés

- ouvrir successivement un écran de chaque métier ;
- passer le téléphone hors connexion puis le reconnecter ;
- vérifier l’apparition du bandeau de mise à jour après un nouveau déploiement ;
- tester les pages Planning Sécurité, Carte Restaurant, Documents Formation et Planning Nettoyage ;
- vérifier que les droits et offres restent inchangés.

## Audit automatique

Le build exécute désormais `node scripts/phase1-static-audit.mjs` avant TypeScript et Vite.
Le contrôle vérifie notamment :

- la cohérence des pages chargées dynamiquement ;
- la compatibilité ES2020 ;
- la présence d’un `search_path` sur les fonctions SQL `SECURITY DEFINER` ;
- la cohérence de la version PWA ;
- les lectures risquées du stockage navigateur.
