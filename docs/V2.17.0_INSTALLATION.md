# NCR Suite V2.17.0 — Formation · BPF automatique

Base requise : **NCR Suite V2.16.0 déployée et validée**.

## Objectif

Cette version prépare le Bilan pédagogique et financier annuel depuis les données Formation déjà présentes. Elle suit la structure du Cerfa 10443*17 sans transformer un calcul incertain en donnée déclarative : les classifications ambiguës restent à vérifier.

## Ordre d’installation

1. Exécuter uniquement `supabase/migrations/077_training_bpf_automation.sql` dans Supabase.
2. Appliquer tous les fichiers du patch V2.17.0 sur GitHub.
3. Redéployer `process-email-queue` si l’Edge Function n’est pas automatiquement publiée depuis GitHub.
4. Lancer le build Cloudflare et vérifier la version `ncr-suite@2.17.0`.
5. Ouvrir l’application, attendre l’activation du cache `ncr-suite-shell-v2.17.0-training-bpf-automation`, puis actualiser la PWA.
6. Contrôler `npm run audit:phase1`, `npm run test:critical`, puis `npm run build`.

Les migrations 001 à 076 étant déjà déployées, elles ne doivent pas être rejouées.

## Points couverts

- Brouillon BPF par exercice comptable.
- Identité de l’organisme, forme juridique, code NAF et signataire.
- Produits HT classés selon le cadre C.
- Chiffre d’affaires et charges du cadre D.
- Formateurs internes et extérieurs du cadre E.
- Publics et heures-stagiaires du cadre F1.
- Activité confiée à un autre organisme du cadre F2.
- Objectifs et niveaux RNCP du cadre F3.
- Cinq principales spécialités du cadre F4.
- Actions reçues en sous-traitance du cadre G.
- Proratisation des heures depuis les émargements.
- Contrôles bloquants et points de vigilance.
- Statuts Brouillon, Vérifié et Verrouillé.
- Exports préparatoires PDF et CSV.
- RLS et contrôle des rôles.

## Vérification fonctionnelle

1. Ouvrir `BPF automatique`.
2. Sélectionner l’exercice à préparer.
3. Ouvrir `Données` et classer les éléments signalés.
4. Ouvrir `Financier`, vérifier l’identité, les produits et les charges.
5. Comparer les cadres F1, F3 et F4 dans `Pédagogique`.
6. Actualiser les calculs et vérifier qu’aucun contrôle bloquant ne subsiste.
7. Marquer le bilan comme vérifié.
8. Exporter le PDF et le CSV.
9. Verrouiller le bilan après rapprochement comptable.

## Limite volontaire

Le PDF produit est un état préparatoire. La déclaration officielle doit être contrôlée et télétransmise dans **Mon Activité Formation**.
