# NCR Suite V2.9.2 — Installation

Cette version s’installe sur la V2.9.1.

## 1. Supabase

Exécuter d’abord :

`supabase/migrations/051_restaurant_stability.sql`

Cette migration ne supprime aucune réservation, table, commande, plat ou donnée existante.

Elle :

- répare l’enregistrement complet des paramètres de réservation Restauration ;
- applique les délais et intervalles de créneaux côté serveur ;
- respecte le mode de confirmation automatique ou manuel ;
- ajoute une protection anti-spam ;
- centralise et audite les changements de statut des réservations.

## 2. GitHub

Ajouter ensuite les autres fichiers du patch en conservant leur arborescence.

## 3. Cloudflare

Vérifier que le commit déclenche un déploiement et que le build termine en vert.

## 4. PWA

Fermer complètement NCR Suite puis la rouvrir.

Cache attendu : `ncr-suite-shell-v2.9.2-restaurant-stable`.

## Tests rapides

1. Modifier dans Paramètres le mode de confirmation, le délai minimum et l’intervalle des créneaux.
2. Recharger les Paramètres et vérifier que les valeurs sont conservées.
3. Ouvrir la page publique et vérifier les limites de dates et les intervalles horaires.
4. Tester une réservation en mode automatique, puis en mode manuel.
5. Installer puis terminer une réservation et vérifier l’état de la table sur le plan de salle.
6. Vérifier les réservations proches de minuit afin de confirmer l’absence de décalage de journée.
