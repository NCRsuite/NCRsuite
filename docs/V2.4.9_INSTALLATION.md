# NCR Suite V2.4.9 — Installation depuis la V2.4.8

Cette mise à jour contient une migration Supabase et un patch GitHub. Respecter impérativement l’ordre ci-dessous.

## 1. Supabase — SQL d’abord

Dans **Supabase → SQL Editor**, ouvrir puis exécuter intégralement :

```text
supabase/migrations/024_training_plan_entitlements.sql
```

Le script doit se terminer sans erreur. Il met à jour les offres Formation, les permissions, le multi-site, la personnalisation, les évaluations et les accès employés.

Ne relancez aucune ancienne migration et ne modifiez pas `process-email-queue`.

## 2. GitHub — patch ensuite

Décompresser le patch V2.4.9 à la racine du dépôt V2.4.8 puis envoyer tout son contenu sur GitHub en conservant l’arborescence.

Accepter le remplacement des fichiers existants et vérifier que les nouveaux fichiers sont bien ajoutés, notamment :

```text
src/pages/TrainingSitesPage.tsx
src/features/training/sessionDossierPdf.ts
supabase/migrations/024_training_plan_entitlements.sql
```

## 3. Cloudflare

Attendre la fin du nouveau déploiement Cloudflare. Le build doit terminer avec le statut **Success**.

## 4. Cache PWA

Après le déploiement :

1. fermer complètement NCR Suite ;
2. rouvrir l’application ;
3. se reconnecter si nécessaire.

Le cache applicatif doit afficher la version V2.4.9.

## 5. Contrôles rapides

Tester avec les formules concernées :

- Découverte : feuille d’émargement vierge, aucune signature numérique ;
- Essentielle : signatures, PDF d’émargement, personnalisation des documents et e-mails ;
- Professionnelle : évaluations, dossier complet, établissements et accès employés avec rôles ;
- Métier : vérifier que les modules configurés sur mesure n’ont pas été réactivés.

Contrôler aussi la rubrique **Mon abonnement**, les convocations et attestations automatiques, puis un envoi Brevo existant.
