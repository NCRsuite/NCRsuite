NCR SUITE V2.13.0 — PATCH UNIQUEMENT
========================================

BASE OBLIGATOIRE
- Installer ce patch uniquement sur la V2.12.3 stable avec le correctif UI PC + mobile validé.
- Ne pas supprimer les fichiers absents du ZIP.
- NCR Academy est un projet séparé et ne doit pas être modifié.

ORDRE D’INSTALLATION
1. Supabase > SQL Editor
   Exécuter intégralement :
   supabase/migrations/065_restaurant_public_menu_premium.sql

2. GitHub
   Copier tous les fichiers du patch en conservant exactement leurs chemins.
   Remplacer les fichiers existants lorsque GitHub le demande.

3. Cloudflare
   Vérifier que le build se termine sans erreur et que la version affichée est 2.13.0.

4. PWA
   Fermer complètement l’application puis la rouvrir afin de charger le cache :
   ncr-suite-shell-v2.13.0-restaurant-premium

AUCUN REDÉPLOIEMENT EDGE FUNCTION
- Cette version ne modifie aucune Edge Function Supabase.

TESTS À EFFECTUER
- Ouvrir Restauration > Personnalisation et enregistrer le nom, les couleurs et un thème.
- Ajouter une couverture et un logo, puis ouvrir le menu public.
- Ajouter ou modifier un plat avec une photo.
- Vérifier les quatre langues FR, EN, ES et IT.
- Vérifier les catégories, allergènes, badges et suggestions du chef.
- Tester le QR code sur iPhone et Android.
- Tester le bouton de réservation si la réservation publique est activée.
- Vérifier le rendu sur petit écran, tablette et ordinateur.
- Vérifier qu’un compte Sécurité peut toujours importer son logo documentaire.

IMPORTANT
- Les anciennes catégories, plats, traductions, réservations et données Restauration sont conservés.
- Le restaurateur reste libre de masquer les photos, allergènes, badges, navigation rapide ou bouton de réservation.
- Les photos sont stockées dans le bucket existant `organization-branding`.
